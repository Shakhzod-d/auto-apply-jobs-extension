# Auto Apply Jobs — Chrome Extension

Manifest V3 extension that autofills (and optionally auto-submits) job
applications on LinkedIn and Indeed, syncing every application and any
unanswered screening questions to the [dashboard](../dashboard) app.

It runs entirely inside your own already-logged-in browser tab — it never
stores or handles your LinkedIn/Indeed credentials.

## Development

```bash
npm install
npm run dev
```

Then in Chrome: `chrome://extensions` → enable Developer mode → **Load
unpacked** → select this project's `dist/` folder (run `npm run build` first,
or `npm run dev` and reload after each change — crxjs HMR handles most
updates automatically).

## Structure

- `manifest.config.ts` — MV3 manifest (generated by `@crxjs/vite-plugin`).
- `src/background/` — service worker; syncs Profile/QuestionBank/Settings
  from the dashboard, coordinates content scripts, opens/navigates the tab a
  bulk run works in.
- `src/content/` — one file per platform (`linkedin.ts`, `indeed.ts`). Each
  handles two things: (1) autofilling any Easy Apply modal that opens,
  whether a human or the bulk driver opened it, and (2) the bulk-apply
  driver loop itself, which works through the current search results page
  clicking Easy Apply on each not-yet-applied listing.
- `src/popup/` — toolbar popup: submit-mode toggle, start/stop the bulk
  run, live progress (processed/submitted/pending/blocked/failed) and
  today's cap usage.
- `src/options/` — settings page: dashboard URL + API token.

## How bulk apply works

1. Set **job search keywords** on the dashboard's Settings page.
2. Click **Start applying on LinkedIn** in the popup. The background worker
   opens (or reuses) a LinkedIn tab and navigates it to a keyword search
   with the Easy Apply filter applied.
3. The content script works through the results list: click a card → click
   Easy Apply → let the fill flow run → pace a random delay (from
   Settings) → next card → next page when the current one is exhausted.
4. If a required question has no matching answer, that application is
   **not** blocked on indefinitely — the question is reported to the
   dashboard, the draft is saved (LinkedIn's own "save for later", not
   discarded), and the loop moves on immediately. Answer it from the
   dashboard's Needs-your-input queue whenever convenient.
5. The loop stops on: daily cap reached (overall or per-platform),
   LinkedIn disabled in Settings, no more results, a submitted-for-review
   application (so it doesn't barrel into the next job out from under you),
   or you clicking **Stop**.

## Safety defaults

Submit mode defaults to **review** (fill the form, stop before submitting)
and pacing/daily caps are enforced from the dashboard's Settings — see the
root plan doc for why. Raise the daily cap and switch to auto-submit only
after you've verified field-matching accuracy on a handful of real
applications.
